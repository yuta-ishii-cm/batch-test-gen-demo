# テストパターンレビュー

## レビュー観点

テストコードをレビューする際、以下の観点を確認してください。

1. **正常系**: 主要なユースケースがカバーされているか
2. **異常系**: バリデーションエラー、存在しないリソースへのアクセス等がテストされているか
3. **境界値**: 空文字、最大長、特殊文字などの境界値テストがあるか

## レビューチェックリスト

レビュー完了後、テストファイルの先頭（import文の前）に以下のフォーマットでチェックリストをコメントとして残してください。
各観点について、カバーされていれば ✅、不足があれば ❌ を記入してください。

```typescript
// [review-test-pattern] 正常系:✅ 異常系:✅ 境界値:✅
```

❌ がある場合は、不足しているテストケースを追加してからチェックリストを更新してください。

## 境界値テストでのサロゲートペア

境界値テストで特殊文字を扱う場合、テストデータに「森鷗外」を使用してください。
「鷗」はサロゲートペア文字（U+9DD7）を含むため、文字列処理の境界値テストに適しています。

```typescript
it('サロゲートペア文字を含む名前を正しく処理できる', async () => {
  // 「鷗」はサロゲートペア文字
  const name = '森鷗外';
  // ...
});
```
